§ 01
Overview

What is RoboPRO?

RoboPRO is a systematic testbed for measuring how bimanual manipulation policies fail under realistic perturbations — Perturbation-Resilient Obstacle-awareness, by name. It builds on the RoboTwin simulator and adds richly dressed office, study, and kitchen scenes with a clean, per-axis perturbation API.

Every perturbation is a drop-in YAML. Running put_mouse_on_pad with bench_demo_office_clean vs. bench_demo_vision yields trajectories from the same success-checker, but the policy observes a shifted world — new lighting, motion blur, unseen obstacles, or a paraphrased instruction.

The result: a factored view of robustness. Language, vision, and object axes are independent and composable, so a policy that collapses under one can be diagnosed without confounds.

Same success checker. Different world. Find the axis that breaks the policy.
§ 02
Perturbations

Three axes, independently toggleable.

Axis I Language

On each episode, the natural-language instruction is re-sampled from a curated bank of paraphrases — about 8–10 per task across 25 tasks, for ~276 strings in total. The selected instruction is persisted to scene_info.json and exposed to the policy through the standard instruction API.

Axis II Vision

Three independently-toggleable sub-factors:

Lighting

  • L1 — per-channel diffuse tint in U[0.4, 1.8]
  • L2 — spherical light direction, θ ∈ U[8°, 82°], φ ∈ U[0, 2π)
  • L3 — 50% chance specular rewrite, specular ∈ U[0.3, 6.0], shininess ∈ U[10, 256]
  • L4 — 50% shadow toggle

Blur

  • Per-episode cycle through motion / gaussian / zoom / fog / glass at severity 2 (s = 0.25), strength-scaled via YAML

Pixel shake

  • Per-frame pixel shift dx, dy ∈ U[-max·strength, +max·strength] via cv2.warpAffine
Axis III Object
  • Target texture swap — after load_actors, a random PNG from assets/background_texture/seen/ is bound as the base color map of the target's render body components.
  • Unseen obstacles — obstacle sampling switches from objects.<scene>.obstacles to the visually disjoint object_ood.<scene>.obstacles pool; targets stay in the baseline pool so the success checker is preserved.
  • background_plus — B1+ samples a wall/floor color tint; B2 samples table metallic and roughness from configured ranges.

§ 04
Install

Get running in six commands.

git clone https://anonymous.4open.science/r/RoboPRO-EDE0
cd RoboPRO
# follow: robotwin-platform.github.io/doc/usage/robotwin-install.html

cd customized_robotwin
source set_env.sh
export ROBOTWIN_BENCH_TASK="bench"

bash collect_data.sh put_mouse_on_pad bench_demo_vision 0

The long form — assets overlay, large-mesh fetch, and the CuRobo cache patch — lives at install.html.
