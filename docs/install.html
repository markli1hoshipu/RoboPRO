Appendix A · Installation

Install.

The long form — six steps from a clean checkout to a first collected episode.

← § Home ↗ Code
§ 1
Clone

Clone.

git clone https://anonymous.4open.science/r/RoboPRO-EDE0
cd RoboPRO
§ 2
Base

RoboTwin base install.

Follow the upstream RoboTwin setup for the conda environment, CuRobo, and SAPIEN: robotwin-platform.github.io/doc/usage/robotwin-install.html. Use the RoboTwin conda environment for all RoboPRO commands.

§ 3
Assets

Assets overlay.

cp -r benchmark/bench_assets/* \
   customized_robotwin/assets/objects_bench/

cp benchmark/bench_assets/embodiments/aloha-agilex/*.yml \
   customized_robotwin/assets/embodiments/aloha-agilex/

Edit curobo_left.yml and curobo_right.yml under customized_robotwin/assets/embodiments/aloha-agilex/ to point at your local URDF paths.

§ 4
Mesh

Large mesh.

The file aloha-agilex/urdf/aloha_maniskill_sim/meshes/box2_Link.dae (~206 MB) is not tracked here. Fetch it from the upstream Aloha-Agilex release and place it at: 

customized_robotwin/assets/embodiments/aloha-agilex/
  urdf/aloha_maniskill_sim/meshes/box2_Link.dae
§ 5
CuRobo

CuRobo cache patch.

In customized_robotwin/envs/curobo/src/curobo/geom/sdf/world_mesh.py, replace the clear_cache method with:

def clear_cache(self):
    self._wp_mesh_cache = {}
    if self._mesh_tensor_list is not None:
        self._mesh_tensor_list[2][:] = 0
    if self._env_n_mesh is not None:
        self._env_n_mesh[:] = 0
    if self._env_mesh_names is not None:
        for i in range(self.n_envs):
            for j in range(len(self._env_mesh_names)):
                self._env_mesh_names[i][j] = None
    super().clear_cache()
§ 6
Verify

Verify.

cd customized_robotwin
source set_env.sh
export ROBOTWIN_BENCH_TASK="bench"

python script/bench_script/visualize_task_scene.py \
    put_mouse_on_pad bench_demo_office_clean --rollout --seed 0

A SAPIEN viewer window should open and play a reference rollout of the task. Once that works, you're ready to collect with any perturbation config.
